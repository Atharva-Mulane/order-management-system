Real-Time Order Management System
This is a full-stack application for managing orders, built with a Spring Boot backend, a React.js frontend, and integrated with AWS cloud services for data storage, file handling, and notifications. This project was built as a full-stack developer assignment.

Live Demo:
Live Frontend Application: http://atharva-order-management-ui.s3-website-us-east-1.amazonaws.com/

Live Backend API (Swagger UI): http://order-management-system-env.eba-9yeskcps.us-east-1.elasticbeanstalk.com/swagger-ui/index.html

Tech Stack:

Backend: Spring Boot, Java 21, Maven 

Frontend: React.js, Vite, Bootstrap, Axios 

Database: AWS DynamoDB 

File Storage: AWS S3 

Notifications: AWS SNS 


Deployment:

Backend: AWS Elastic Beanstalk

Frontend: AWS S3 Static Website Hosting


CI/CD: GitHub Actions 

Features

Dashboard: View a list of all orders in a clean, responsive table. 

Create Orders: A dedicated form to create new orders with customer name, amount, and a PDF invoice upload. 

Order Details: View detailed information for a single order.

Cloud Integration:

Order data is stored securely in DynamoDB. 

Invoice PDF files are uploaded to S3, and the URL is stored. 

Real-time notifications are sent via SNS upon order creation. 


API Documentation: Interactive API documentation available via Swagger UI. 


CI/CD: Automated build and test pipeline using GitHub Actions. 

AWS Setup Instructions 

To run this project with your own AWS resources, you must configure the following:

IAM User: Create an IAM user with programmatic access and attach policies granting full access for S3, DynamoDB, and SNS.

S3 Bucket (for Invoices):

Create an S3 bucket.

On the Permissions tab, disable "Block all public access" and save the changes.

DynamoDB Table:

Create a DynamoDB table named Orders.

Set the Partition key to orderId with the type String.

SNS Topic:

Create a standard SNS Topic.

Create an Email subscription to this topic using your email address.

Confirm the subscription by clicking the link in the confirmation email you receive.

How to Run Locally:

Backend (order-service)
Navigate to Directory: cd order-service

Configure Properties:

Create a file at src/main/resources/application.properties.

Add your AWS credentials, region, S3 bucket name, and SNS Topic ARN to this file. Note: This file is listed in .gitignore and should not be committed.

Run the Application:

mvnw spring-boot:run
The backend will be available at http://localhost:8080.

Frontend (order-ui)
Navigate to Directory: cd order-ui

Install Dependencies:


npm install
Update API URL:

In src/services/orderService.js, change the API_URL constant to http://localhost:8080/orders.

Run the Application:

npm run dev
The frontend will be available at http://localhost:5173.

API Reference 

API documentation is generated by Swagger and is available once the backend is running locally.
